## 3. 風險詞自動檢查流程

收到使用者 prompt 時，依序檢查：

1. **掃描紅色詞**：任一命中 → 拒絕，提供安全替代方向（SKILL.md §25）
2. **掃描組合詞**：發現高風險組合 → 拒絕或重寫其中一邊
3. **掃描黃色詞**：每個都套上中和詞或改寫
4. **掃描繞過意圖**：發現「為了避審 / 限界突破 / 回避策 / 怎樣才能讓 AI 畫色情」→ 觸發反繞過聲明（SKILL.md §27）
5. **確認綠色詞密度**：最終 prompt 應有 8+ 個綠色詞，建立明確的「editorial / commercial / refined」脈絡

---

## 4. 為什麼這套詞庫有效

gpt-image-2 的審查不是關鍵字比對，而是**整體意圖判讀**。一個 prompt 是「色情」還是「editorial」，由整段語意密度決定：

- 8 個綠色詞 + 1 個黃色詞（已中和）→ 通常通過 → 高品質出圖
- 3 個綠色詞 + 3 個黃色詞（未中和）→ 邊緣，可能擋、可能廉價輸出
- 任何紅色詞 / 高風險組合 → 直接擋、或強行輸出但畫面廉價油膩

「安全寫法」的本質**不是藏詞**，而是**讓模型確信你要的是 editorial 而不是色情**。模型分不清「高級性感」和「擦邊」——它只忠實反映 prompt 的語意密度。

## 5. 完整負面 Constraints 模板（複製即用）

每個寫真 prompt 結尾建議貼上以下完整 Constraints 段（依場景挑選）：

### 通用寫真完整負面

```text
Constraints: Do not stylize the face. Do not cartoonize. No anime. No CGI look. No illustration. No childlike features. No teen styling. No school uniform. No fantasy armor. No nudity. No lingerie styling. No exaggerated anatomy. No body-part close-up. No low-angle body gaze. No voyeuristic angle. No undressing gesture. No clothing slipping off. No plastic skin. No over-smoothing. No beauty retouching. No AI artifacts. No oversharpened edges. No deformed hands. No extra fingers. No missing fingers. No fused fingers. No broken anatomy. No facial distortion. No asymmetric eyes. No melted features. No fabric warping. No broken straps. No pattern misalignment. No jewelry distortion. No watermark. No logo. No extra text. No duplicate text. No frame border.
```

### Reference Image 模式追加

```text
Change only the wardrobe, pose, lighting, background, and camera framing. Do not redesign the character. Do not change the face shape, facial proportions, eye spacing, nose bridge length, mouth width, jawline, head size, hairstyle, hair color, or identity. Keep everything else listed above the same.
```

### 3D CG 模式追加

```text
Premium commercial illustration quality. No real-camera realism. No documentary look. No photographic skin pores. Refined material response. Detailed hair strands. Soft global illumination.
```

## 6. 拒絕回覆範本

當使用者請求觸發紅色詞 / 反繞過聲明，使用以下模板：

```text
這個方向我無法協助。本 Skill 拒絕任何 [指出具體類型：未成年性化 / 名人冒名 / jailbreak / 多人親密場景 / 繞過審查] 的請求。

我可以幫你做的安全替代方向：
- [建議 1：例如「同樣的氛圍，改成 editorial 雜誌風」]
- [建議 2：例如「同樣的人物，改成商業 Lookbook 風」]
- [建議 3：例如「同樣的場景，改成 cinematic 自然光肖像」]

需要哪個方向？或你希望調整哪一個元素？
```

絕對不要：
- 解釋「如何繞過」
- 給「降低 ban 率」建議
- 把性化內容包裝成「藝術詞」後輸出
- 用 reference image 做名人冒名 / 未成年 / 性化生成
